Setup in 3 steps

No API keys, no coding, no app installation required.

1

Get a Slack Incoming Webhook URL

Open the Incoming Webhooks app on the Slack Marketplace.

Select your workspace in the upper right corner, then click Add to Slack.

Choose the channel where you want to receive Expandi notifications and click Add Incoming WebHooks integration.

Copy the Webhook URL you see on the next page (starts with https://hooks.slack.com/services/...).

2

Generate your Expandi webhook endpoint

Paste the Slack Webhook URL you just copied below. We'll generate a unique endpoint URL that connects Expandi to your Slack channel.

Click to copy · Use this URL in Expandi's webhook settings (Step 3)
3

Add the webhook in Expandi

In the Expandi app, configure your webhook to start receiving notifications:

  1. Go to LinkedIn Settings → Webhooks
  2. Click Add a webhook
  3. Paste the generated endpoint URL from Step 2
  4. Select the event: Contact replied to campaign message
  5. Save — you're done!

Test the connection

Questions

Is this free?
Yes, completely free. There are no limits on the number of notifications.
Do I need to install a Slack app?
Just the free Incoming Webhooks app from the Slack Marketplace. Click "Add to Slack", pick a channel, and you'll get a Webhook URL — that's all you need.
Can I send notifications to multiple channels?
Yes. Create a separate Slack webhook for each channel, generate a separate endpoint here, and add each one in Expandi's webhook settings. For example: replies go to #sales-replies, connections go to #new-connections.
Which Expandi events can I get notified about?
All 27 Expandi webhook events are supported — replies, connections, messages sent, profile visits, email opens, and more. In Expandi, go to LinkedIn Settings → Webhooks to see the full list. Add one webhook per event type.
How fast are the notifications?
Near real-time. When Expandi fires the webhook, the notification appears in Slack within 1-2 seconds.
Is my data secure?
We don't store any of your data. The webhook payload flows through our relay server to your Slack channel and nothing is saved. Your Slack webhook URL is encoded in the endpoint URL itself — no database, no accounts, no data retention.
Can I use this for multiple LinkedIn accounts?
Yes. The same endpoint URL works across all your Expandi profiles — just paste it into the webhook settings of each LinkedIn account. Every notification shows which account sent it (the "Sender" field), so you'll always know who triggered it. If you'd prefer separate channels per account, generate a different endpoint for each Slack channel.
